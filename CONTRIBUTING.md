# Contributing to AgentForge

Thanks for contributing.

## Development flow
1. Fork the repository.
2. Create a feature branch.
3. Test `bin/agentforge` and `bin/agentforge-console` locally.
4. Update docs when behavior changes.
5. Open a pull request with screenshots or terminal captures for UI changes.

## Standards
- Keep the console fast and dependency-light.
- Prefer standard library implementations.
- Preserve compatibility with local terminal environments.
- Document any new stage, skill, or command.

## Pull request checklist
- [ ] Command-line flow tested
- [ ] README updated if needed
- [ ] No local-only paths committed
- [ ] Screenshots/GIF attached for console UI changes
